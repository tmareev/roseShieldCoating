# RoseShield Coatings

Адаптивный одностраничный сайт компании по нанесению полимочевинных покрытий на бетонные полы и стены в Чикаго и пригородах.

## Что реализовано

- Большой hero-блок с призывами к действию и ключевыми преимуществами.
- Разделы услуг, преимуществ polyurea-покрытия, процесса работ, отзыва и FAQ.
- Галерея проектов с фильтрами `All`, `Garage`, `Commercial` и `Basement`.
- Просмотр фотографии проекта в полноэкранном lightbox.
- Форма запроса бесплатной оценки с HTML-валидацией и экраном успешной отправки.
- Адаптивное меню и отдельная нижняя панель действий на мобильных устройствах.
- Плавная навигация по секциям страницы.
- Адаптивная верстка для desktop, tablet и mobile.
- Глобальное UI-состояние на Redux Toolkit.

> Важно: сейчас форма работает только на клиенте. Она показывает успешную отправку, но не передает данные на сервер, в CRM или на email. Телефон, email, отзывы и ссылки социальных сетей используются как демонстрационный контент и должны быть заменены перед публикацией.

Проверить установленные версии:

```bash
node --version
npm --version
```

## Быстрый запуск

Из корня проекта:

```bash
npm install
npm run dev
```

После запуска откройте адрес, который Vite напечатает в терминале. Обычно это:

```text
http://localhost:5173/
```

Dev-сервер запущен с параметром `--host 0.0.0.0`, поэтому сайт также можно открыть с другого устройства в той же локальной сети по сетевому адресу компьютера, который Vite покажет как `Network`.

Чтобы остановить сервер, нажмите `Ctrl+C` в терминале, где он запущен.

## Основные команды

```bash
# Установить зависимости из package-lock.json
npm install

# Запустить dev-сервер с автоматическим обновлением
npm run dev

# Проверить TypeScript и собрать production-версию
npm run build

# Локально посмотреть уже собранную production-версию
npm run preview
```

Production-сборка создается в папке `dist/`. Для ее предварительного просмотра сначала выполните:

```bash
npm run build
npm run preview
```

Обычно preview-сервер будет доступен по адресу `http://localhost:4173/`.

## Структура проекта

```text
.
├── src/
│   ├── assets/          # Изображения сайта
│   ├── App.tsx          # Секции страницы и интерактивные компоненты
│   ├── main.tsx         # Точка входа React, Bootstrap и Redux Provider
│   ├── store.ts         # Redux store, фильтр галереи и статус формы
│   └── styles.css       # Общие стили и адаптивная верстка
├── index.html           # HTML-шаблон Vite
├── package.json         # Зависимости и npm-команды
├── vite.config.ts       # Конфигурация Vite
└── tsconfig*.json       # Конфигурация TypeScript
```

## Где менять контент

- Тексты, контакты, услуги, проекты, FAQ и поля формы: `src/App.tsx`
- Изображения: `src/assets/`
- Цвета, типографика, размеры и responsive-стили: `src/styles.css`
- Логика фильтра галереи и статуса отправки формы: `src/store.ts`

Основные цвета вынесены в CSS-переменные в начале `src/styles.css`:

```css
:root {
  --ink: #29272c;
  --pink: #d98fa6;
  --pink-dark: #b96480;
  --paper: #f8f7f7;
}
```

## Подготовка к реальному запуску

Перед размещением сайта в интернете рекомендуется:

1. Заменить демонстрационные телефон, email, отзывы и социальные ссылки.
2. Подключить форму к API, CRM, email-сервису или serverless-функции.
3. Добавить обработку ошибок и состояние загрузки формы.
4. Проверить юридические тексты, гарантийные обещания и контакты компании.
5. Оптимизировать изображения в `src/assets/` для уменьшения размера загрузки.
6. Добавить favicon, Open Graph metadata, аналитику и SEO-описание.

## Чистая переустановка зависимостей

Если после обновления зависимостей проект работает некорректно:

```bash
rm -rf node_modules
npm install
npm run dev
```
Для стопа сервера используем
```bash
ctrl+С
```

Файл `package-lock.json` следует хранить в Git: он фиксирует совместимые версии зависимостей.
